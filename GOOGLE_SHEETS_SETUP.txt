Google Sheets live sync setup

1. Create a new Google Sheet.
2. Open Extensions > Apps Script.
3. Paste the contents of Code.gs from this bundle.
4. Save the project.
5. Deploy > New deployment > Web app.
6. Execute as: Me
7. Who has access: Anyone with the link
8. Copy the Web App URL.
9. In admin.html, open Google Sheets Sync tab.
10. Paste the URL and click Save URL.
11. Click Test Connection.
12. Click Save Entire League to Google Sheets.
13. Public page and player page will read live data automatically from Google Sheets.

Reset behavior
- In admin.html > Google Sheets Sync, click Reset Google Sheets Data.
- This clears the stored league state in Google Sheets so you can start another league later.



Automatic DUPR-style leaderboard from Google Sheets

This version calculates the season leaderboard and DUPR-style ratings inside Apps Script.
That means:
- Google Sheets stores the league state
- Apps Script calculates the leaderboard when data is loaded or saved
- Public page reads the computed leaderboard automatically
- A Leaderboard tab is also written into your Google Sheet


Public score entry

This version allows score updates directly from public.html for current active scheduled matches only. Scores are validated by both the public page and Apps Script before being saved into Google Sheets.


Important after this update

If you see "Unsupported POST action" when saving scores from public.html, your deployed Apps Script is still using an older Code.gs version.

Fix:
1. Open your Google Apps Script project
2. Replace Code.gs with the new Code.gs from this bundle
3. Save the project
4. Deploy > Manage deployments
5. Edit your Web App deployment
6. Deploy the new version
7. Refresh public.html

After redeploying, Save Score on public.html will update Google Sheets successfully.


Admin auto-refresh

This version refreshes admin.html from Google Sheets every 15 seconds when a valid Apps Script URL is configured. That means scores saved from public.html will appear in admin automatically after a short delay.


Public week sign-up

This version adds a Week Sign-Up tile to public.html. Players can sign up for the current active week separately for Saturday and Sunday. The sign-up is saved into Google Sheets availability, and admin schedule generation uses the same availability list.


Important for Week Sign-Up

If players were seeing Failed to fetch while saving sign-up, update Code.gs from this bundle and redeploy your Apps Script Web App. This version supports updateWeekSignup through GET and hides sign-up once matches are generated or completed.


Current Signup Week

Admin can now select a Current Signup Week. Public sign-up uses that week first. Schedule generation in admin also checks that players have signed up for the selected week/day before generating matches.


If Week Sign-Up shows 'Unsupported GET action', replace Code.gs with this version and redeploy your Apps Script Web App. This bundle also includes a POST fallback in public.html.


Week Setup auto-sync

This version auto-syncs Week Setup changes to Google Sheets. When you toggle Saturday or Sunday active, or change courts/rounds, the admin page now pushes those changes automatically if the Google Apps Script URL is configured.
